## 🗂️ 项目结构

```
kiro-travel/
├── app/                    # Next.js App Router
│   ├── api/               # API 路由
│   ├── (auth)/            # 认证相关页面
│   └── (main)/            # 主要功能页面
├── components/            # React 组件
│   ├── ui/               # UI 基础组件
│   └── ...               # 业务组件
├── contexts/              # React Context
│   └── auth-context.tsx  # 认证上下文
├── lib/                   # 工具库
│   ├── db.ts            # SQLite 数据库配置
│   ├── db-utils.ts      # 数据库操作工具
│   ├── auth.ts          # 认证工具
│   └── api-client.ts    # API 请求工具
├── data/                  # SQLite 数据库文件
│   └── travel.db        # 本地数据库
├── scripts/              # 脚本文件
│   ├── init-sqlite.mjs  # 数据库初始化
│   └── setup.mjs        # 项目初始化
├── types/                # TypeScript 类型定义
└── docs/                 # 文档目录（当前目录）
```

## 🔧 技术栈

### 前端
- **框架**: Next.js 16 + React 19
- **语言**: TypeScript
- **样式**: Tailwind CSS 4
- **UI 组件**: Radix UI
- **状态管理**: React Context
- **表单**: React Hook Form + Zod

### 后端
- **运行时**: Next.js API Routes
- **数据库**: SQLite + better-sqlite3
- **认证**: JWT (jsonwebtoken)
- **密码加密**: bcryptjs

## 📝 开发命令

```bash
# 一键初始化（推荐）
npm run setup

# 手动初始化
npm install          # 安装依赖
npm run db:init     # 初始化数据库

# 开发服务器
npm run dev         # 启动开发服务器

# 构建和部署
npm run build       # 构建生产版本
npm run start       # 启动生产服务器

# 数据库操作
npm run db:reset    # 重置数据库
```

## ❓ 常见问题

**Q: 数据库存储在哪里？**

A: 数据库文件位于 `data/travel.db`，这是一个 SQLite 文件数据库。

**Q: 如何添加新的 API 端点？**

A: 在 `app/api/` 目录下创建新的路由文件，参考现有 API 的实现。

**Q: 如何修改数据库表结构？**

A: 修改 `scripts/001_create_tables.sql` 文件，然后运行 `npm run db:reset` 重置数据库。

**Q: 普通用户和导游的区别？**

A:
- **普通用户**: 可以浏览、评论、收藏、购买门票、预订酒店
- **导游**: 可以创建和管理景点、查看游客信息
- **管理员**: 拥有所有权限，可以管理所有数据
