# 09. 部署指南

本文档详细说明了旅游服务系统的本地部署和生产环境部署方案。

## 🏠 本地部署

### 系统要求

- **Node.js**: 18.0 或更高版本
- **操作系统**: Windows、macOS、Linux
- **内存**: 至少 2GB RAM
- **磁盘空间**: 至少 1GB 可用空间

### 快速部署步骤

1. **克隆项目**
```bash
git clone <your-repo-url>
cd kiro-travel
```

2. **一键初始化**
```bash
npm run setup
```

3. **启动服务**
```bash
npm run dev
```

4. **访问应用**
- 前端: http://localhost:3000
- API: http://localhost:3000/api

### 手动部署步骤

1. **安装依赖**
```bash
npm install
```

2. **配置环境变量**
```bash
cp .env.example .env.local
```

编辑 `.env.local`:
```env
# JWT 密钥（生产环境必须更改）
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production

# 数据库路径（可选）
DB_PATH=./data/travel.db
```

3. **初始化数据库**
```bash
npm run db:init
```

4. **启动开发服务器**
```bash
npm run dev
```

### 开发环境命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器 |
| `npm run build` | 构建生产版本 |
| `npm run start` | 启动生产服务器 |
| `npm run lint` | 代码检查 |
| `npm run db:init` | 初始化数据库 |
| `npm run db:reset` | 重置数据库 |

## 🚀 生产环境部署

### 部署选项

#### 1. Vercel 部署（推荐）

**优点**:
- 零配置部署
- 自动 HTTPS
- 全球 CDN
- 自动扩容

**部署步骤**:

1. **准备项目**
```bash
# 安装 Vercel CLI
npm i -g vercel

# 登录 Vercel
vercel login
```

2. **配置环境变量**
在 Vercel 控制台添加环境变量：
- `JWT_SECRET`: 强密钥（至少 32 字符）
- `NODE_ENV`: `production`

3. **部署项目**
```bash
vercel --prod
```

**注意事项**:
- SQLite 数据库只读，不能用于写入操作
- 如需写入功能，需使用外部数据库服务

#### 2. Docker 部署

**创建 Dockerfile**:
```dockerfile
# Dockerfile
FROM node:18-alpine

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production

# 复制源代码
COPY . .

# 构建应用
RUN npm run build

# 创建数据目录
RUN mkdir -p data

# 暴露端口
EXPOSE 3000

# 启动应用
CMD ["npm", "start"]
```

**创建 docker-compose.yml**:
```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - JWT_SECRET=${JWT_SECRET}
    volumes:
      - ./data:/app/data
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - app
    restart: unless-stopped
```

**部署命令**:
```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f
```

#### 3. PM2 部署

**安装 PM2**:
```bash
npm install -g pm2
```

**创建 PM2 配置文件**:
```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'travel-system',
    script: 'npm',
    args: 'start',
    cwd: './',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'development'
    },
    env_production: {
      NODE_ENV: 'production',
      JWT_SECRET: 'your-production-secret'
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_file: './logs/combined.log',
    time: true
  }]
}
```

**部署脚本**:
```bash
#!/bin/bash
# deploy.sh

# 安装依赖
npm ci

# 构建项目
npm run build

# 使用 PM2 启动
pm2 start ecosystem.config.js --env production

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

#### 4. 传统服务器部署

**系统要求**:
- Ubuntu 20.04+ / CentOS 8+
- Nginx
- Node.js 18+
- PM2 (可选)

**部署步骤**:

1. **安装 Node.js**
```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
```

2. **安装和配置 Nginx**
```bash
sudo apt update
sudo apt install nginx

# 创建配置文件
sudo nano /etc/nginx/sites-available/travel-system
```

Nginx 配置:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

3. **启用站点**
```bash
sudo ln -s /etc/nginx/sites-available/travel-system /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

4. **部署应用**
```bash
# 克隆代码
git clone <your-repo-url>
cd kiro-travel

# 安装依赖
npm ci

# 构建项目
npm run build

# 初始化数据库
npm run db:init

# 使用 PM2 启动
pm2 start npm --name "travel-system" -- start
```

## 🔒 HTTPS 配置

### 使用 Let's Encrypt

1. **安装 Certbot**
```bash
sudo apt install certbot python3-certbot-nginx
```

2. **获取证书**
```bash
sudo certbot --nginx -d your-domain.com
```

3. **自动续期**
```bash
sudo crontab -e
# 添加
0 12 * * * /usr/bin/certbot renew --quiet
```

### Nginx HTTPS 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 3. 健康检查

**创建健康检查端点**:
```typescript
// app/api/health/route.ts
import { NextResponse } from 'next/server'
import { db } from '@/lib/db'

export async function GET() {
  try {
    // 检查数据库连接
    db.prepare('SELECT 1').get()

    return NextResponse.json({
      status: 'healthy',
      timestamp: new Date().toISOString()
    })
  } catch (error) {
    return NextResponse.json(
      {
        status: 'unhealthy',
        error: error.message
      },
      { status: 503 }
    )
  }
}
```

## 🔄 备份策略

### 1. 数据库备份

**创建备份脚本**:
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/home/user/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_PATH="/home/user/travel-system/data/travel.db"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp $DB_PATH $BACKUP_DIR/travel_$DATE.db

# 压缩备份
gzip $BACKUP_DIR/travel_$DATE.db

# 删除 7 天前的备份
find $BACKUP_DIR -name "travel_*.db.gz" -mtime +7 -delete

echo "Backup completed: travel_$DATE.db.gz"
```

**设置定时备份**:
```bash
sudo crontab -e
# 添加
0 2 * * * /home/user/backup.sh
```

### 2. 代码备份

```bash
#!/bin/bash
# deploy-backup.sh

APP_DIR="/home/user/travel-system"
BACKUP_DIR="/home/user/backups/code"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建代码备份
tar -czf $BACKUP_DIR/code_$DATE.tar.gz -C $APP_DIR .

echo "Code backup completed: code_$DATE.tar.gz"
```

## 🚨 故障排查

### 1. 常见问题

**问题：端口被占用**
```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>
```

**问题：数据库锁定**
```bash
# 检查是否有其他进程使用数据库
lsof data/travel.db

# 删除锁文件
rm data/travel.db-wal data/travel.db-shm
```

**问题：内存不足**
```bash
# 查看内存使用
free -h

# 查看进程内存使用
ps aux --sort=-%mem | head
```

### 2. 日志分析

```bash
# 查看错误日志
tail -f logs/err.log

# 查看访问日志
tail -f /var/log/nginx/access.log

# 分析日志统计
grep "ERROR" logs/combined.log | wc -l
```

### 3. 性能分析

```bash
# 安装性能监控工具
npm install -g clinic

# 运行性能分析
clinic doctor -- node server.js
```
