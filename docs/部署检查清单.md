## 🚀 Render 部署步骤

### 1. 创建 Render 账号

- [ ] 访问 https://render.com/
- [ ] 使用 GitHub 账号登录（推荐）
- [ ] 授权 Render 访问 GitHub

### 2. 创建 Web Service

- [ ] 点击 Dashboard 中的 "New +" → "Web Service"
- [ ] 选择 `kiro-travel` 仓库
- [ ] 点击 "Connect"

### 3. 配置服务

#### 基本设置
- [ ] **Name**: `kiro-travel`（或你想要的名称）
- [ ] **Region**: `Singapore` 或 `Oregon`（选择离你近的）
- [ ] **Branch**: `master`
- [ ] **Root Directory**: 留空
- [ ] **Runtime**: `Node`
- [ ] **Build Command**: `npm install && npm run build`
- [ ] **Start Command**: `npm start`

#### 实例类型
- [ ] **Instance Type**: 选择 `Free`

### 4. 配置持久化磁盘

- [ ] 向下滚动找到 "Disks" 部分
- [ ] 点击 "Add Disk"
- [ ] **Name**: `data`
- [ ] **Mount Path**: `/data`
- [ ] **Size**: `1 GB`
- [ ] 点�� "Save"

### 5. 配置环境变量

点击 "Advanced" 展开，添加以下环境变量：

- [ ] `NODE_ENV` = `production`
- [ ] `DATABASE_PATH` = `/data/database.sqlite`
- [ ] `JWT_SECRET` = 点击 "Generate" 生成随机值
- [ ] `NEXT_PUBLIC_APP_URL` = `https://your-service-name.onrender.com`
  - ⚠️ **重要**：先用占位符，部署后再更新为实际 URL

### 6. 开始部署

- [ ] 检查所有配置无误
- [ ] 点击页面底部 "Create Web Service"
- [ ] 等待部署完成（约 6-11 分钟）

## 🐛 故障排查

### 构建失败

**症状**：Logs 中显示 "Build failed"

**检查项**：
- [ ] `package.json` 中的 `build` 脚本是否正确
- [ ] 本地运行 `npm run build` 是否成功
- [ ] Node 版本是否兼容（推荐 18.x）

**解决方案**：
```bash
# 在 Render 环境变量中添加
NODE_VERSION=18.17.0
```

### 启动失败

**症状**：构建成功但服务无法启动

**检查项**：
- [ ] `package.json` 中的 `start` 脚本是否正确
- [ ] 端口配置是否正确（Next.js 会自动使用 `PORT` 环境变量）
- [ ] 数据库路径是否正��

**解决方案**：
检查启动日志中的错误信息，通常会指示具体问题。

### 数据库错误

**症状**：网站打开但提示数据库错误

**检查项**：
- [ ] 持久化磁盘是否正确配置
- [ ] `DATABASE_PATH` 环境变量是否设置
- [ ] 数据库文件是否存在

**解决方案**：
```bash
# 在 Render Shell 中运行
echo $DATABASE_PATH  # 确认环境变量
ls -la /data/         # 检查文件是否存在
npm run db:init       # 重新初始化数据库
```

### 数据丢失

**症状**：重新部署后数据消失

**原因**：未配置持久化磁盘，或数据库路径不在 `/data` 目录

**解决方案**：
- [ ] 确认已配置 Disk（Settings → Disks）
- [ ] 确认 `DATABASE_PATH=/data/database.sqlite`
- [ ] 重新初始化数据库

## 📊 性能优化（可选）

### 减少构建时间

- [ ] 在 `package.json` 中只保留生产依赖
- [ ] 将开发工具移到 `devDependencies`

### 优化数据库

- [ ] 添加数据库索引（在 `init-db.js` 中）
  ```javascript
  db.exec(`
    CREATE INDEX IF NOT EXISTS idx_tickets_spot_id ON tickets(spot_id);
    CREATE INDEX IF NOT EXISTS idx_orders_user_id ON orders(user_id);
  `);
  ```

### 启用缓存

- [ ] 在 `next.config.mjs` 中启用：
  ```javascript
  const nextConfig = {
    output: 'standalone',
    compress: true,
  }
  ```
