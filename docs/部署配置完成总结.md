## 🚀 快速部署步骤

### 第一步：提交代码到 GitHub

```bash
# 查看修改
git status

# 添加所有文件
git add .

# 提交
git commit -m "配置 Render 部署支持

- 添加 render.yaml 配��文件
- 修改数据库代码支持持久化存储
- 添加健康检查 API
- 配置 GitHub Actions 自动部署
- 完善部署文档"

# 推送到 GitHub
git push origin master
```

### 第二步：部署到 Render

1. 访问 https://render.com/ 并用 GitHub 登录
2. 点击 "New +" → "Web Service"
3. 选择 `kiro-travel` 仓库
4. Render 会自动读取 `render.yaml` 配置
5. 点击 "Create Web Service"
6. 等待部署完成（6-11 分钟）

### 第三步：初始化数据库

部署完成后，在 Render Shell 中运行：

```bash
npm run db:init
```

### 第四步：设置 UptimeRobot

1. 访问 https://uptimerobot.com/ 注册
2. 创建监控器，URL 设为你的 Render 服务地址
3. 监控间隔设为 5 分钟

### ✅ 零代码改动部署
- 数据库代码自动适配本地和 Render 环境
- 无需手动修改连接字符串

## 🆘 故障排查

### 常见问题

#### 1. 部署失败

**症状**：构建过程中断
**排查**：
```bash
# 本地测试构建
npm run build

# 检查 Node 版本
node --version  # 应该是 18.x
```

#### 2. 数据库连接失败

**症状**：网站打开提示数据库错误
**排查**：
```bash
# 在 Render Shell 中
echo $DATABASE_PATH  # 应该是 /data/database.sqlite
ls -la /data/        # 检查文件是否存在
```

#### 3. 服务一直休眠

**症状**：每次访问都需要等待冷启动
**排查**：
- 检查 UptimeRobot 监控是否正常运行
- 确认监控间隔为 5 分钟
- 查看 UptimeRobot 日志确认 ping 成功
